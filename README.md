# Codex 使用技巧知识库

这个仓库用于记录 Codex 在真实项目中的使用技巧、可复用流程和踩坑经验。

## 初心

这个仓库的初心，是把一次次和 Codex 协作时真正有效的做法沉淀下来。

很多技巧不是抽象教程里学来的，而是在真实任务里跑出来的：让 Codex 控制本地命令行、登录远程服务器、自动验证部署结果、整理操作步骤、补齐文档、最后形成可复用流程。这样的经验如果只留在聊天记录里，很快就会散掉；写进仓库后，下次遇到相似任务就能直接复用。

这个仓库不追求收集所有提示词，也不追求写成大而全的教程。它更像一个个人工程手册：记录什么场景适合让 Codex 自动做，怎么给它边界，怎么验证结果，哪些坑下次不要再踩。

重点沉淀：

- Codex 如何控制本地命令行、远程服务器、浏览器或其他工具完成验证。
- 哪些任务适合自动化，哪些任务应该保留人工确认。
- 可复用的提示词、执行步骤和检查清单。
- 真实案例中的问题、修复和验证证据。

## 目录

```text
cases/      真实案例复盘
patterns/   可复用操作模式
tools/      工具能力说明
```

## 已记录案例

- `cases/vps-deploy-automation.md`：让 Codex 自动验证 VPS 部署脚本，包括 SSH、SFTP、远程命令、服务状态检查和文档沉淀。

## 记录原则

- 写可复用的步骤，不只写聊天结论。
- 记录验证证据，例如命令、状态、退出码和产物路径。
- 不记录真实密码、Token、私钥、客户端配置等敏感信息。
- 对未验证的平台明确标注“未实测”。
