# 远程服务器自动化验证模式

适用于让 Codex 验证 VPS、云服务器、远程 Linux 环境上的部署脚本。

## 推荐流程

1. 先确认本地仓库状态。
2. 如果远程服务器要从 GitHub 拉代码，先确认最新提交已 push。
3. 登录远程服务器前，检查是否有自动化 SSH 工具，例如 Paramiko。
4. 远程执行前先检查 clean state：

```bash
systemctl is-active 服务名
test -e /目标/安装路径
ss -ltnp | grep ':端口'
```

5. 先跑 dry-run。
6. 再跑正式部署。
7. 最后验证服务、端口、配置和产物。

## 验证证据

至少记录：

- OS 和 CPU 架构。
- 克隆到的 Git commit。
- dry-run 是否通过。
- 正式部署是否通过。
- 服务是否 active。
- 端口是否 LISTEN。
- 配置测试是否 OK。
- 客户端或部署产物是否存在。

## 安全边界

- 不把密码写入仓库。
- 不提交生成的客户端配置、私钥、Token。
- 自动化失败时，先检查远程是否半安装，避免重复部署造成误判。

## 适合自动化的任务

- 重复部署验证。
- 脚本兼容性验证。
- 服务状态检查。
- 端口监听检查。
- 部署产物下载验证。

## 不建议完全自动化的任务

- 购买域名。
- 开放云厂商安全组。
- 处理账号、支付、实名认证。
- 长期保存 SSH 密码或云平台 Token。
