# 仓库维护规则

这个仓库是给我自己看的 Codex 使用技巧知识库。

写东西时要简单、直接、好懂。不要写成教程论文，也不要写太多抽象概念。

## 这个仓库记录什么

只记录我真实用过、以后可能还会复用的 Codex 技巧。

每个技巧单独一个文件，放在：

```text
tips/
```

例如：

```text
tips/paramiko-ssh-login.md
```

## 每个技巧的固定格式

每个技巧都按这三个部分写：

```text
原因
解决
效果
```

意思是：

- 原因：为什么需要这个技巧，原来哪里不方便。
- 解决：具体怎么做，给出能复制的命令。
- 效果：用了之后解决了什么问题，能带来什么结果。

可以额外加：

```text
注意
展望
```

但不要把文件写得太长。

## 写作风格

- 用普通人能听懂的话。
- 少用术语；必须用术语时，顺手解释一下。
- 命令要能复制执行。
- 不要只写结论，要写“为什么这样做”。
- 不要写空话，例如“提升效率”“增强能力”，除非后面说明具体提升在哪里。

## 不要记录什么

不要记录任何敏感信息：

- 真实密码
- Token
- 私钥
- 客户端代理配置
- 完整连接链接
- 云厂商账号信息

如果需要举例，用假值：

```text
服务器 IP: x.x.x.x
用户名: root
密码: 只在当前任务中使用
```

## 提交前检查

提交前至少检查：

```powershell
git status --short
git diff --check
```

确认没有把敏感文件、临时文件、虚拟环境提交进去。
